**Purpose:** Global state that must outlive any single page.

## 4. Data Flow

### REST API (projects, tags, contact, admin)
```
Page renders → TanStack Query hook fires
  → checks cache (staleTime 5 min)
  → if stale/absent: lib/api/*.ts called
  → src/lib/api/client.ts (Axios) attaches Bearer token
  → NestJS API at VITE_API_BASE_URL
  → ApiResponse<T>.data unwrapped → typed result
  → TanStack Query caches; component re-renders
```

### Firebase Chat (real-time, event-driven)
```
User sends message
  → useChatRoom.sendMessage() or useAnonymousChat.addMessage()
  → Firebase push() to RTDB path
  → Firebase broadcasts to ALL connected clients simultaneously
  → onValue() listener fires in every mounted hook instance
  → setMessages() → component re-renders
  (no polling — purely push-based)
```

### Two separate chat systems
| System | Hook | Firebase path | Behaviour |
|--------|------|--------------|-----------|
| Anonymous bot chat | `useAnonymousChat` | `chat/messages` | Floating bubble on HomePage; bot auto-replies; admin can clearMessages() (deletes for ALL users) |
| Live chat room | `useChatRoom` | `chatroom/messages` | Full page (/chat) + preview strip on HomePage; real visitors; no clear function |

### Time Blocking (fully offline)
```
User creates/edits block
  → useTimeBlocking state update
  → useEffect fires synchronously after render
  → lib/timeblock.saveSchedule() → localStorage['timeblocking-schedules']
  (no server, no network)
```

### PokéAPI (Pokédex — no key required)
```
PokedexPage mounts
  → usePokemonListInfinite (useInfiniteQuery) fetches pokeapi.co/api/v2/pokemon?offset=0&limit=20
  → each PokemonCard mounts → usePokemonDetail fetches /pokemon/<name> (stale 24 h, cached in TanStack Query)
  → type filter selected → usePokemonByType fetches /type/<type> (stale 10 min)
  → search filters displayNames client-side across already-loaded results
```

### Bangkok AQI (WAQI API — demo token, no env var)
```
BangkokAQIPage mounts
  → useAQIData fires fetchData() immediately
  → native fetch → api.waqi.info/feed/bangkok/?token=demo
  → maps response to AQIData · getAQILevel(aqi) selects tier from AQI_LEVELS
  → setInterval(fetchData, 5 min) auto-refreshes
  → ParticleCanvas adjusts count/speed/colour from AQILevel
```

### RPS Game (fully offline)
```
User selects hero
  → HEROES loaded via import.meta.glob at build time (no runtime fetch)
  → Battle loop: player choice → randomChoice() for bot → resolveRound() → timed state transitions
  → Streak: read/write to localStorage['rps-streak'] on game over
```

### Static Fallback Toggle
```
VITE_API_BASE_URL set?
  YES → Axios REST calls for projects, tags, admin
  NO  → STATIC_PROJECTS / STATIC_TAGS returned immediately

VITE_TAT_API_KEY set?
  YES → TAT API fetch; on any error → silently returns STATIC_ATTRACTIONS
  NO  → STATIC_ATTRACTIONS returned immediately
```

| Key | Written by | Format | Risk if corrupted |
|-----|-----------|--------|-----------------|

## 10. Known Gotchas & Non-Obvious Patterns

- **Static fallback is evaluated at module load, not runtime.** `!import.meta.env.VITE_API_BASE_URL` runs once when the JS module is imported. Changing `.env` requires restarting the dev server — hot reload is not enough.

- **`lib/api/thailand.ts` and `lib/api/pokedex.ts` use native `fetch`, not Axios.** Both APIs must never receive an `Authorization` header. TAT API is proxied via `/tat-api`; PokéAPI goes direct to `pokeapi.co`. Using `apiClient` for either would attach a JWT and cause failures.

- **`clearMessages()` deletes for ALL visitors.** `useAnonymousChat.clearMessages()` calls `remove(ref(db, 'chat/messages'))` which immediately deletes every message globally across all connected clients.

- **Two Firebase paths — they do not share data.** `chat/messages` (anonymous bot, floating bubble) and `chatroom/messages` (live room, ChatPage + ChatPreviewSection) are entirely separate. A message sent in one never appears in the other.

- **`ChatPreviewSection` no longer exists.** The old index referenced it, but the file `src/components/home/ChatPreviewSection.tsx` is absent. `HomePage` no longer renders a live-chat preview strip.

- **`/admin/messages` route does not exist.** `AdminLayout` renders a nav link to `/admin/messages` but this route is not defined in `App.tsx`. Clicking it renders `NotFoundPage`.

- **Admin lazy chunk + `RequireAuth` ordering.** `RequireAuth` reads `AuthContext.isAuthenticated` synchronously from localStorage before the lazy chunk loads — the redirect to login happens before the JS bundle is even requested.

- **`usePublishProject` / `useUnpublishProject` invalidate the public cache.** Both mutations call `qc.invalidateQueries({ queryKey: queryKeys.projects.all })` in addition to the admin list — so public-facing `HomePage` and `ProjectsPage` will refetch after any admin publish/unpublish action.

- **No debounce on TimeBlocking saves.** `saveSchedule()` and `saveBlockTypes()` are triggered by `useEffect` on every state change. High-frequency drag interactions can cause many synchronous localStorage writes.

- **`useAQIData` does NOT use TanStack Query.** It manages its own `useState`/`setInterval` polling. There is no cache, no `queryKeys` entry, and no `queryClient.invalidateQueries` path. Refreshing is done via the returned `retry()` callback.

- **WAQI `demo` token is rate-limited.** The AQI hook hardcodes `token=demo` in its API URL. This token may return stale data or be throttled under load. Replace with a real key before any production deployment.

- **`HEROES` is resolved at Vite build time via `import.meta.glob`.** All SVGs in `src/data/hero/` are bundled as assets. Adding a file requires a dev-server restart (or rebuild) to appear. The hero roster order is alphabetical by filename.

- **RPS phase transitions use `setTimeout`, not Framer Motion lifecycle.** The battle logic in `RpsPage` uses nested `setTimeout` chains to sequence HP drain and gameover detection. Avoid reducing the delays without testing the full reveal sequence visually.

- **`WelcomeOverlay` session detection is undocumented.** The mechanism used to detect a first-time visit is not clear from the surrounding code. Investigate before changing.

- **`Province.id` must match TopoJSON `NAME_1` exactly.** `ThailandMap.tsx` matches province data by comparing `Province.id` against the `NAME_1` field in `public/data/thailand-provinces.json`. Any mismatch causes a province to render as unclickable on the map.

- **Manual chunk split in `vite.config.ts`.** vendor/query/motion are split into separate JS bundles. Admin pages are further split via `React.lazy`. Do not import admin-only heavy dependencies into public-facing pages.
